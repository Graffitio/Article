문서 업로드 전략부터 심층 분석까지

Practical Guide · RAG · Prompting · 2026 · by Ian Jung

언어 모델(LLM) 활용의 성패는 결국 '소스(Source)'에 달려있습니다. "그냥 요약해 줘" 식의 뻔한 프롬프트와 철저하게 통제된 소스 기반의 대화는 차원이 다른 결과를 만들어냅니다. 모든 RAG(검색 증강 생성) 환경에서 소스 기반 대화가 최우선시되어야 하는 이유이며, NotebookLM은 바로 이 지점에 특화된 강력한 도구입니다.

1. 대화 스타일 정의 : 페르소나와 구조 부여하기

단순히 문서를 던져주고 질문하는 것을 넘어, AI가 어떤 렌즈를 통해 문서를 바라볼지 대화 스타일을 정의해야 합니다. 가장 효과적이고 흔히 쓰이는 페르소나는 비판적이고 분석적인 '애널리스트(Analyst)'와 맥락을 짚어주며 설명하는 '가이드(Guide)'입니다.

대화 스타일을 설정할 때는 단순히 역할만 부여하는 것이 아니라, 상황(목적) 부여와 답변 구조(서론-본론-결론 등)를 나누어 명확한 컨텍스트(Context)를 제공해야 합니다.

UI 셋팅 가이드 : 맞춤 설정(Custom Instructions) 적용

채팅창 내 시스템 프롬프트 설정 기능(맞춤 설정 ⚙️ 버튼)을 활용하여 AI의 기본 행동 지침을 세팅하세요. 매 질문마다 번거롭게 배경 설명을 할 필요 없이, 사전에 정의된 규칙에 따라 답변을 출력하게 됩니다.

대화 스타일 설정 예시

구분 설정 내용 (프롬프트 예시) 기대 효과
상황 부여
(목적 & 페르소나)		 "너는 10년 차 IT 산업 애널리스트야. 이 문서에서 향후 3년간의 시장 리스크 요인을 찾아 경영진에게 보고할 목적이야." 일반론적인 답변을 배제하고, 실무적이고 날카로운 비즈니스 관점의 인사이트 도출
답변 구조
(출력 포맷)		 "답변은 항상 [핵심 요약(1문단)] - [근거 3가지(불릿 포인트)] - [결론 및 제언] 구조로 작성해." 가독성이 극대화되며, 복잡한 문서에서도 원하는 정보만 빠르게 스캐닝 가능

2. 전략적인 문서 업로드 : 30페이지의 법칙

수백 페이지의 책이나 방대한 논문을 통째로 업로드하는 것은 피해야 합니다. 현재 대형 언어 모델이 컨텍스트 내에서 정보를 누락 없이 가장 잘 읽어내는 한계점은 대략 20~30페이지 분량입니다.

따라서 문서를 의미 있는 단위나 챕터별로 쪼개어 업로드하는 것이 핵심입니다. 주제별 + 30페이지 단위로 분할하여 소스를 제공하면 AI의 정보 추출 능력이 비약적으로 상승합니다.

문서를 무작정 쏟아붓지 마십시오. 잘게 쪼갠 20~30페이지 단위의 소스들을 마인드맵 구조로 인식하고 업로드하는 것만으로도, AI의 정보 인식률과 연결성은 압도적으로 올라갑니다.

3. Depth가 높은 질문 : 메모 기반의 선순환

NotebookLM의 진정한 파괴력은 일회성 채팅이 아닌 '메모(Notes)' 기능에서 나옵니다. 대화를 통해 유의미한 답변을 얻어내고, 이를 메모로 저장한 뒤, 그 메모들만을 다시 소스(Source)로 선택하여 질문의 뎁스(Depth)를 높이는 방식이 가장 강력합니다.

여러 소스에 파편화되어 있던 정보들을 모아 새로운 결론을 내거나, 복잡한 개념 간의 마인드맵을 형성하는 데 이보다 뛰어난 방법은 없습니다.

심층 분석을 위한 3단계 메모 스킬

Step 01 · 1차 개념 질문 (개별 챕터)
업로드한 책이나 보고서에서 특정 부분을 공부할 때, 먼저 개별 개념에 대한 질문을 던집니다. (예: "A라는 개념의 정의와 한계는 무엇인가?")
Step 02 · 핵심 답변 노트 저장 (Save to Note)
도출된 양질의 답변 중 모든 챕터에 사용되는 보편적이고 중요한 개념들을 '메모(Note)'로 핀(Pin)하여 저장합니다.
Step 03 · 메모 기반의 결론 도출 및 마인드맵
저장된 여러 개의 메모들을 선택(Select)하여 집중 소스로 설정한 뒤, "이 메모들을 종합하여 새로운 결론을 내줘" 혹은 "개념 간의 연결성을 마인드맵 구조로 그려줘"라고 요청합니다.

핵심 용어 정리

소스 기반 대화 (Source-Grounded Conversation)
Information Grounding

AI가 학습한 방대한 사전 지식에 의존하는 대신, 사용자가 직접 제공한 문서(소스) 내에서만 사실을 확인하고 답변을 생성하도록 강제하는 방식입니다. 환각(Hallucination) 현상을 최소화합니다.

RAG (검색 증강 생성)
Retrieval-Augmented Generation

질문과 관련된 외부 지식이나 문서를 검색(Retrieval)하여 가져온 뒤, 이를 바탕으로 AI가 답변을 생성(Generation)하는 기술입니다. NotebookLM은 개인화된 RAG 시스템의 훌륭한 예시입니다.

컨텍스트 윈도우 한계
Context Window Limit

AI가 한 번에 처리하고 기억할 수 있는 텍스트의 양입니다. 기술적으로는 수백 페이지를 넣을 수 있지만, 집중도(Attention)가 분산되지 않고 고품질의 정보 추출을 보장하는 실질적 한계는 20~30페이지 내외로 보는 것이 실무적 관례입니다.
