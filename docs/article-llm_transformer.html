LLM의 기본 원리
: 트랜스포머

트랜스포머 아키텍처의 핵심 원리 —
어텐션 메커니즘부터 LLM 학습까지

Technical Essay · Transformer · Attention · 2026 · by Ian Jung

트랜스포머(Transformer)는 2017년 Google 연구팀이 번역을 위해 제안한 신경망 구조다. 그러나 오늘날 이것은 ChatGPT, Claude, Gemini 같은 대형 언어 모델 전체의 심장이 되었다. 어떻게 하나의 아이디어가 AI 혁명을 이끌었는가.

트랜스포머 이전 — 순서대로 읽던 시대

트랜스포머가 등장하기 전, 자연어 처리의 주류는 RNN(순환 신경망, Recurrent Neural Network)이었다. RNN은 문장을 왼쪽에서 오른쪽으로 한 단어씩 처리한다. 마치 책을 한 글자씩 손으로 짚어가며 읽는 것과 같다.

이 방식에는 두 가지 치명적 약점이 있었다. 첫째, 문장이 길어질수록 앞에서 읽은 내용을 잊어버린다 — 장기 의존성 문제(Long-term Dependency Problem). 둘째, 단어를 반드시 순서대로 처리해야 하므로 병렬화가 불가능해 학습이 느렸다.

장기 의존성 문제 (Long-term Dependency Problem)

긴 문장에서 앞쪽의 정보가 뒤쪽 처리에 제대로 전달되지 않는 현상. "나는 어제 도서관에서 책을 빌렸는데, 그 책의 저자가 쓴 또 다른 작품이…"에서 RNN은 "그 책"이 무엇인지 기억하기 어렵다. 트랜스포머의 어텐션이 이를 근본적으로 해결했다.

어텐션 — "어디에 집중할지"를 계산한다

트랜스포머의 혁신은 단순하다. 단어를 하나씩 순서대로 읽는 대신, 문장 전체를 한꺼번에 보면서 단어 간 관련도를 동시에 계산한다. 이것이 어텐션(Attention) 메커니즘이다.

사람이 긴 글을 읽을 때 모든 단어에 동등하게 주의를 기울이지 않는 것처럼, 어텐션은 각 단어가 다른 단어와 얼마나 관련 있는지를 수치로 계산해 중요한 단어에 더 집중하도록 만든다.

이 계산에는 세 가지 벡터가 사용된다. Query(Q)는 "나는 무엇을 찾고 있나"를, Key(K)는 "나는 어떤 정보를 갖고 있나"를, Value(V)는 "실제로 전달할 정보"를 담는다. Q와 K를 비교해 관련도 점수를 계산하고, 그 점수에 따라 V를 가중합산한다.

어텐션의 핵심은 병렬성이다. RNN이 단어를 시간 순서로 처리하는 동안, 어텐션은 문장의 모든 단어 쌍 관계를 동시에 계산한다. 이것이 트랜스포머를 빠르게, 그리고 강하게 만든 이유다.

멀티헤드 어텐션 — 여러 눈으로 동시에

멀티헤드 어텐션(Multi-Head Attention)은 어텐션을 여러 개 병렬로 수행하는 방식이다. 각 "헤드(head)"는 서로 다른 관점에서 단어 간 관계를 포착한다. 하나의 헤드가 문법적 관계를 보는 동안, 다른 헤드는 의미적 유사성을, 또 다른 헤드는 대명사-명사 지시 관계를 분석한다.

각 헤드의 결과를 합치면 단 하나의 어텐션으로는 불가능한 풍부한 언어 이해가 완성된다. 이것이 트랜스포머가 복잡한 언어 패턴을 정교하게 포착하는 핵심 이유다.

Q / K / V 벡터의 역할

벡터 역할 비유
Query (Q) 나는 무엇을 찾고 있나? 검색어
Key (K) 나는 어떤 정보를 갖고 있나? 책의 목차
Value (V) 실제로 전달할 정보 책의 본문

구조 — 인코더와 디코더

트랜스포머는 크게 인코더(Encoder)와 디코더(Decoder)로 나뉜다. 인코더는 입력 문장의 의미를 압축된 표현으로 변환하고, 디코더는 그 표현을 받아 출력 문장을 생성한다.

오늘날의 대형 언어 모델은 용도에 따라 구조가 다르다. BERT는 인코더만 사용해 문장 이해와 분류에 강하고, GPT와 Claude는 디코더만 사용해 텍스트 생성에 특화되어 있다. 번역·요약 모델은 인코더-디코더 구조 전체를 활용한다.

한 가지 중요한 보완이 있다. 트랜스포머는 문장 전체를 동시에 처리하므로 단어의 순서 정보가 자동으로 반영되지 않는다. 이를 보완하기 위해 포지셔널 인코딩(Positional Encoding)을 별도로 계산해 각 단어의 위치 정보를 주입한다.

트랜스포머 아키텍처의 흐름

01 · Input Embedding
텍스트를 토큰으로 분리하고, 각 토큰을 고차원 벡터로 변환한다. 포지셔널 인코딩을 더해 순서 정보를 주입.
02 · Self-Attention
모든 단어 쌍의 관련도를 동시에 계산. Q·K·V 행렬 연산으로 문맥을 반영한 표현이 생성된다.
03 · Multi-Head Attention
여러 헤드가 다양한 관점에서 병렬로 어텐션을 수행. 결과를 합쳐 풍부한 문맥 표현을 완성.
04 · Feed-Forward + Norm
어텐션 결과를 피드포워드 네트워크로 추가 변환. 레이어 정규화로 학습을 안정화.
05 · Output Generation
디코더(또는 언어 모델 헤드)가 가장 확률 높은 다음 토큰을 반복 생성하며 텍스트를 완성한다.

LLM 학습 — "다음 단어 맞히기"의 반복

LLM은 트랜스포머 구조를 바탕으로 방대한 텍스트를 학습한다. 학습 방식은 놀랍도록 단순하다. "다음 단어를 맞혀라."

"오늘 날씨가 매우 ___"라는 문장에서 빈칸을 맞히는 훈련을 수조 번 반복하면서, 모델은 언어의 패턴, 문법, 사실, 논리적 관계까지 자연스럽게 학습한다. 이 과정에서 수십억~수천억 개에 달하는 파라미터(Parameter)가 점점 정교하게 조정된다.

트랜스포머는 "언어를 이해한다"기보다 "언어의 패턴을 극도로 정교하게 모델링한다"고 보는 것이 정확하다. 그럼에도 그 결과는 놀랍도록 사람과 유사한 텍스트 생성 능력으로 이어진다.

왜 트랜스포머는 이토록 강력한가

  • 병렬 처리. 모든 단어를 동시에 처리하므로 RNN 대비 학습 속도가 월등히 빠르다. GPU/TPU의 병렬 연산 능력을 최대로 활용한다.
  • 장거리 의존성 해결. 어텐션은 문장 내 어떤 두 단어 사이의 관계도 동일한 비용으로 계산한다. 거리에 관계없이 관련 정보를 포착한다.
  • 확장성(Scalability). 데이터와 파라미터를 늘릴수록 성능이 꾸준히 향상된다. "더 크면 더 좋다"는 스케일링 법칙이 성립하는 구조다.
  • 범용성. 언어, 이미지, 음성, 코드 — 순서 있는 데이터라면 무엇이든 처리할 수 있다. 이것이 트랜스포머가 AI 전 분야를 지배하게 된 이유다.

용어 정리

LLM
Large Language Model · 대형 언어 모델

방대한 텍스트 데이터로 학습된 AI 언어 모델. ChatGPT, Claude, Gemini 등이 해당된다.

트랜스포머 (Transformer)
Vaswani et al., 2017

어텐션 메커니즘을 기반으로 한 신경망 구조. 현재 대부분의 LLM의 핵심 구조다.

RNN
Recurrent Neural Network · 순환 신경망

단어를 순서대로 처리하는 구조. 장기 의존성 문제와 병렬화 불가로 한계가 있었다.

셀프 어텐션 (Self-Attention)
Scaled Dot-Product Attention

같은 문장 내 단어들 사이의 관계를 계산하는 어텐션. Q·K·V 행렬 연산으로 구현된다.

멀티헤드 어텐션 (Multi-Head Attention)
Multi-Head Attention

여러 관점에서 병렬로 어텐션을 수행하는 기법. 다양한 언어 패턴을 동시에 포착한다.

포지셔널 인코딩 (Positional Encoding)
Positional Encoding

단어의 순서 정보를 모델에 주입하는 방법. 트랜스포머가 병렬 처리를 하므로 위치 정보를 별도로 추가해야 한다.

파라미터 (Parameter)
Parameter

모델이 학습을 통해 조정하는 내부 수치값. LLM은 수십억~수천억 개의 파라미터를 가진다.

장기 의존성 문제
Long-term Dependency Problem

긴 문장에서 앞쪽 정보를 유지하지 못하는 RNN의 한계. 트랜스포머의 어텐션이 이를 해결한다.
